The Office-31 benchmark dataset has 4,110 images spread across three domains, capturing objects under varying conditions:

Amazon (Web catalog) 2,817 images (68.5%)
Webcam (Low-res captures) 795 images (19.3%)
DSLR (High-res camera) 498 images (12.2%)

Setup Hint: The DSL works natively with this folder layout. Write scripts to convert, scale down to ViT-friendly dimensions (96x96 or 32x32), and export metadata to clean up dirty data beforehand.

DSL Preprocessing Grammar

We built a domain-specific scripting engine using Lark to compile declarative statements into image processing sequences:

count dataset "path" Counts files by subfolder/class hierarchy.
verify dataset "path" corrupt Tests file integrity using Pillow open/verify.
resize dataset "path" to 32x32, 96x96 outdir "out" Lanczos-downscales to multiple outputs.
convert dataset "path" to rgb outdir "out" Converts color models (rgb, grayscale).
detail dataset "path" output "report.csv" Generates resolution & shape spreadsheets.

Catastrophic Forgetting in Vision Transformers

When training Vision Transformers (ViT) sequentially on different domains (e.g., first Amazon images, then Webcam, and finally DSLR), a severe drop in validation accuracy occurs on the early domains. This is known as **Catastrophic Forgetting** or the **Stability-Plasticity Dilemma**.

  • Plasticity: The model's capacity to acquire new knowledge from webcam and DSLR domains.
  • Stability: The model's capacity to preserve previous catalog features learned from Amazon.

Theoretical Background

Why do ViTs forget?

Unlike Convolutional Neural Networks (CNNs) which exhibit structural spatial inductive biases (locality and translation equivariance), Vision Transformers must learn relationships between patch projections purely through data. When fine-tuned on target tasks, the self-attention projection weights ($\mathbf{W}_Q, \mathbf{W}_K, \mathbf{W}_V$) shift. Without restriction, they overwrite patterns tuned to previous visual representations.

Forgetting Mitigation Candidates

Method Type Pros Cons
Elastic Weight Consolidation (EWC) Regularization Prevents parameter drift via Fisher Information matrix constraint. Computes and stores second-order derivatives; high memory overhead.
Replay Buffer Memory-based Stores exemplars from past tasks for concurrent training. Requires extra storage; risks privacy violations on datasets.
Prompt Pooling (L2P) Architecture Locks main weights, updates small prompt keys inside attention loops. Slightly slower inference; difficult initialization.
LoRA Adapters Parameter-Efficient Keeps base ViT parameters frozen; allocates tiny rank-decomposition matrices. Multi-task routing requires dynamic attention redirection.

Research Focus: In Vision Transformers, the linear projection layer right before the attention blocks and the MLP layers are the highest contributors to forgetting. Mitigation methods must focus attention on these areas.
